# PRD — 제품 요구사항 정의서

## 1. 제품 개요

| 항목 | 내용 |
|---|---|
| 제품명 | 칸반 보드 (Kanban Board) |
| 버전 | 2.0 |
| 작성일 | 2026-05-20 |
| 작성자 | ksl3011 |

### 한 줄 설명

할 일(To-Do) · 진행 중(In-Progress) · 완료(Done) 세 컬럼으로 구성된 경량 태스크 관리 도구.  
Google 또는 GitHub 계정으로 로그인하면 카드를 드래그해 상태를 변경하고, 어느 기기에서든 데이터가 동기화된다.

---

## 2. 배경 및 목적

개인·소규모 팀이 복잡한 협업 도구(Jira, Trello 등) 없이 **단순한 태스크 흐름을 시각화**하고 싶을 때 활용한다.  
v2.0에서는 OAuth 로그인을 추가해 기기 간 카드 동기화와 데이터 보호를 제공한다.

---

## 3. 사용자

| 사용자 유형 | 설명 |
|---|---|
| 개인 개발자 | Google 또는 GitHub 계정으로 로그인해 작업 목록을 관리 |
| 소규모 팀 | 각자 계정으로 로그인해 자신의 보드를 관리 |
| 학습자 | OAuth·Supabase 연동 패턴 학습 목적 |

---

## 4. 핵심 기능

### 4.1 인증

| ID | 기능 | 우선순위 |
|---|---|---|
| F-06 | Google 계정으로 로그인 (OAuth 2.0 / PKCE) | Must |
| F-07 | GitHub 계정으로 로그인 (OAuth 2.0 / PKCE) | Must |
| F-08 | 로그아웃 | Must |
| F-09 | 미인증 상태에서 보드 접근 차단 (로그인 화면으로 리다이렉트) | Must |

### 4.2 카드 관리

| ID | 기능 | 우선순위 |
|---|---|---|
| F-01 | 컬럼별 카드 추가 (텍스트 입력) | Must |
| F-02 | 카드 삭제 | Must |
| F-03 | 카드 드래그 앤 드롭으로 컬럼 간 이동 | Must |
| F-04 | 컬럼 헤더에 카드 수 배지 표시 | Should |
| F-05 | 카드 데이터 Supabase DB에 영속 저장 (기기 간 동기화) | Must |

### 4.3 UX 세부 요구사항

- **F-06/07 OAuth**: 버튼 클릭 → 해당 플랫폼 인증 페이지 → 복귀 시 자동으로 보드 표시
- **F-08 로그아웃**: 헤더의 로그아웃 버튼 클릭 → 세션 삭제 → 로그인 화면
- **F-09 인증 게이트**: 페이지 로드 시 세션 확인 → 없으면 로그인 화면, 있으면 보드
- **F-01 카드 추가**: "+ 카드 추가" 버튼 클릭 → textarea 폼 → Enter(줄바꿈 아님) 또는 확인 버튼으로 저장 → Escape 또는 취소 버튼으로 폼 닫기. 빈 텍스트 시 폼 유지
- **F-03 드래그 앤 드롭**: 드래그 중 카드 반투명 처리, 유효 드롭존 진입 시 시각 피드백(배경색·점선 테두리)

---

## 5. 비기능 요구사항

| 항목 | 요구사항 |
|---|---|
| 브라우저 지원 | Chrome / Firefox / Safari / Edge 최신 2버전 |
| 빌드 도구 | 없음 — 정적 HTML·CSS·JS 파일 |
| 외부 의존성 | Supabase JS SDK v2 (CDN) |
| 인증 방식 | Supabase Auth — OAuth 2.0 PKCE 플로우 |
| 성능 | 카드 100개 이하에서 드래그 응답 16ms 이내 |
| 접근성 | 버튼에 `title` 속성 제공 (스크린리더 기본 지원) |
| 데이터 격리 | Supabase RLS — 본인 카드만 읽기·쓰기 가능 |

---

## 6. 범위 외 (Out of Scope)

- 다중 보드 지원
- 카드 우선순위·마감일·담당자 필드
- 카드 순서 변경 (같은 컬럼 내 재정렬)
- 실시간 협업 (여러 사용자가 같은 보드를 동시 편집)
- 모바일 터치 드래그
- 이메일/비밀번호 자체 회원가입

---

## 7. 성공 지표

| 지표 | 목표값 |
|---|---|
| 페이지 로드 후 첫 인터랙션까지 | < 1초 |
| OAuth 로그인 완료 후 보드 표시까지 | < 2초 |
| 카드 이동 응답 시간 | < 100ms (UI 반영) |
| Supabase CRUD 실패 시 사용자 알림 | 즉시 |
