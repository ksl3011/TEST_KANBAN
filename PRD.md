# PRD — 제품 요구사항 정의서

## 1. 제품 개요

| 항목 | 내용 |
|---|---|
| 제품명 | 칸반 보드 (Kanban Board) |
| 버전 | 1.0 |
| 작성일 | 2026-05-20 |
| 작성자 | ksl3011 |

### 한 줄 설명

할 일(To-Do) · 진행 중(In-Progress) · 완료(Done) 세 컬럼으로 구성된 경량 태스크 관리 도구.  
별도 백엔드·회원가입 없이 브라우저만으로 즉시 사용 가능하며, 카드를 드래그해 상태를 변경한다.

---

## 2. 배경 및 목적

개인·소규모 팀이 복잡한 협업 도구(Jira, Trello 등) 없이 **단순한 태스크 흐름을 시각화**하고 싶을 때 활용한다.  
설치·로그인·서버 없이 HTML 파일 하나를 열면 바로 동작하는 것이 핵심 가치이다.

---

## 3. 사용자

| 사용자 유형 | 설명 |
|---|---|
| 개인 개발자 | 자신의 작업 목록을 가볍게 관리 |
| 소규모 팀 | 개인 PC에서 보드를 열고 현황 공유(화면 공유 등) |
| 학습자 | 칸반 방법론 이해·실습 목적 |

---

## 4. 핵심 기능 (MVP)

### 4.1 카드 관리

| ID | 기능 | 우선순위 |
|---|---|---|
| F-01 | 컬럼별 카드 추가 (텍스트 입력) | Must |
| F-02 | 카드 삭제 | Must |
| F-03 | 카드 드래그 앤 드롭으로 컬럼 간 이동 | Must |
| F-04 | 컬럼 헤더에 카드 수 배지 표시 | Should |
| F-05 | 페이지 새로고침 후 상태 유지 (localStorage) | Must |

### 4.2 UX 세부 요구사항

- **F-01 카드 추가**: "+ 카드 추가" 버튼 클릭 → textarea 폼 표시 → Enter(줄바꿈 아님) 또는 확인 버튼으로 저장 → Escape 또는 취소 버튼으로 폼 닫기
- **F-03 드래그 앤 드롭**: 드래그 중 카드 반투명 처리, 유효 드롭존 진입 시 시각 피드백(배경색·점선 테두리)
- **F-05 영속성**: 카드 추가·삭제·이동 직후 즉시 localStorage에 저장

---

## 5. 비기능 요구사항

| 항목 | 요구사항 |
|---|---|
| 브라우저 지원 | Chrome / Firefox / Safari / Edge 최신 2버전 |
| 빌드 도구 | 없음 — 순수 HTML·CSS·JS |
| 외부 의존성 | 없음 (CDN 포함 불가) |
| 성능 | 카드 100개 이하에서 드래그 응답 16ms 이내 |
| 접근성 | 버튼에 `title` 속성 제공 (스크린리더 기본 지원) |

---

## 6. 범위 외 (Out of Scope)

- 사용자 인증 / 다중 사용자 협업
- 다중 보드 지원
- 카드 우선순위·마감일·담당자 필드
- 카드 순서 변경 (같은 컬럼 내 재정렬)
- 서버 동기화 / 실시간 업데이트
- 모바일 터치 드래그

---

## 7. 성공 지표

| 지표 | 목표값 |
|---|---|
| 페이지 로드 후 첫 인터랙션까지 | < 1초 |
| 카드 이동 응답 시간 | < 100ms |
| localStorage 저장 실패율 | 0% (정상 환경 기준) |
